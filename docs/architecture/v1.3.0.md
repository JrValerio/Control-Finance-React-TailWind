# Arquitetura v1.3.0 (Foundation)

## Objetivo
Estabelecer base monorepo para evoluir autenticacao e persistencia em backend sem quebrar o frontend atual.

## Estrutura

```text
apps/
  web/  -> Vite + React (UI, testes de fluxo)
  api/  -> Express (healthcheck, base para auth/transactions)
```

## Fluxo atual

```text
Browser (apps/web)
   |
   | GET /health
   v
API Express (apps/api)
   |
   v
JSON: { ok: true, version: "1.3.0" }
```

## Decisoes
- Monorepo com npm workspaces para reduzir friccao operacional.
- Root scripts orquestram `dev/lint/test/build` para os dois apps.
- API inicia por `/health` para validar rede, env e deploy antes de auth/CRUD.
- Service layer no web (`src/services`) prepara transicao de localStorage para backend.
