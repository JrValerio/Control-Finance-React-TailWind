# Arquitetura v1.6.5 (Web TypeScript Thin Slice)

## Objetivo
Iniciar migracao incremental para TypeScript no `apps/web` sem reescrever o app inteiro.

## Escopo

- Setup minimo de TypeScript no workspace web
- Script de validacao de tipos (`typecheck`)
- Conversao do service `transactions.service` de `.js` para `.ts`
- Tipagem do contrato paginado (`data + meta`) e payloads de transacao

## Mudancas

### Configuracao (`apps/web`)
- `tsconfig.json` com migracao gradual (`allowJs: true`, `checkJs: false`)
- `tsconfig.node.json` para tooling local
- `vite-env.d.ts`
- Script `typecheck` em `apps/web/package.json`

### Service tipado
- `src/services/transactions.service.ts`:
  - tipos de dominio (`Transaction`, `TransactionType`)
  - tipos de listagem (`TransactionListOptions`, `PaginationMeta`, `TransactionsPageResult`)
  - tipos de payload (`TransactionCreatePayload`, `TransactionUpdatePayload`)

## Resultado

- Base pronta para migracao incremental de outros modulos para TypeScript
- Contratos de listagem paginada mais explicitos no frontend
- Sem alteracao de comportamento de runtime
